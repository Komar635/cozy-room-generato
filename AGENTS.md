# Правила работы с Memory Bank

Этот проект использует систему Memory Bank для поддержания контекста между сессиями.

## Файлы Memory Bank
- **projectbrief.md**: Основные цели и задачи проекта.
- **productContext.md**: Почему этот проект существует, пользовательские проблемы и решения.
- **activeContext.md**: Текущий фокус, активные задачи и ближайшие шаги.
- **systemPatterns.md**: Архитектурные решения, паттерны проектирования и технический дизайн.
- **techContext.md**: Технологический стек, инструменты и ограничения.
- **progress.md**: История изменений, список выполненных задач и метрики.

## Правила обновления
1. Обновлять Memory Bank в конце каждой значимой итерации или по запросу пользователя.
2. Следить за тем, чтобы `activeContext.md` всегда отражал реальное состояние дел.
3. Каждое крупное техническое решение должно фиксироваться в `systemPatterns.md`.
4. В `progress.md` обязательно вести хронологию изменений.
5. При каждом новом сеансе проверять актуальность данных.
