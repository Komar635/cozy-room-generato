# Пошаговая настройка Yandex Cloud

## ✅ ВЫ ЗДЕСЬ: Консоль Yandex Cloud открыта

### Шаг 1: Создание PostgreSQL (5 минут)

1. В левом меню найдите **"Managed Service for PostgreSQL"** (иконка с базой данных)
2. Нажмите **"Создать кластер"**
3. Заполните форму:

**Базовые настройки:**
- Имя: `reality-digitizer-db`
- Окружение: `PRODUCTION`
- Версия: `PostgreSQL 15`

**Вычислительные ресурсы:**
- Класс хоста: `s2.micro` (2 vCPU, 8 GB RAM) - для начала хватит
- Размер хранилища: `10 GB`

**База данных:**
- Имя БД: `reality_digitizer`
- Имя пользователя: `app_user`
- Пароль: придумайте надежный (сохраните его!)

**Сеть:**
- Оставьте настройки по умолчанию

4. Нажмите **"Создать кластер"**
5. ⏳ Подождите 5-10 минут пока кластер создается

### Шаг 2: Получение данных подключения

После создания кластера:

1. Откройте созданный кластер
2. Перейдите на вкладку **"Обзор"**
3. Найдите раздел **"Подключение"**
4. Скопируйте:
   - **Хост** (например: `c-c9qash4696kqj8e7hbmg.rw.mdb.yandexcloud.net`)
   - **Порт** (обычно `6432`)

### Шаг 3: Создание Object Storage (2 минуты)

1. В левом меню найдите **"Object Storage"**
2. Нажмите **"Создать бакет"**
3. Заполните:
   - Имя: `reality-digitizer-storage` (должно быть уникальным!)
   - Класс хранилища: `Стандартное`
   - Доступ: `Ограниченный`
4. Нажмите **"Создать бакет"**

### Шаг 4: Создание сервисного аккаунта (3 минуты)

1. В левом меню найдите **"Service Accounts"** (Сервисные аккаунты)
2. Нажмите **"Создать сервисный аккаунт"**
3. Заполните:
   - Имя: `reality-digitizer-sa`
   - Роли: 
     - Нажмите "Добавить роль" → выберите `storage.editor`
     - Нажмите "Добавить роль" → выберите `mdb.admin`
4. Нажмите **"Создать"**

### Шаг 5: Создание ключей доступа (2 минуты)

1. Откройте созданный сервисный аккаунт
2. Перейдите на вкладку **"Статические ключи доступа"**
3. Нажмите **"Создать ключ доступа"**
4. **⚠️ ВАЖНО**: Сохраните оба значения:
   - **Идентификатор ключа** (Access Key ID)
   - **Секретный ключ** (Secret Access Key)
   
   ⚠️ Секретный ключ показывается только один раз!

### Шаг 6: Обновление .env.local

Откройте файл `.env.local` в вашем проекте и замените значения:

```env
# PostgreSQL (из Шага 2)
DATABASE_URL=postgresql://app_user:ВАШ_ПАРОЛЬ@c-xxx.rw.mdb.yandexcloud.net:6432/reality_digitizer?sslmode=require

# Object Storage (из Шага 5)
YC_STORAGE_ENDPOINT=https://storage.yandexcloud.net
YC_STORAGE_ACCESS_KEY=ваш_access_key_id
YC_STORAGE_SECRET_KEY=ваш_secret_key
YC_STORAGE_BUCKET=reality-digitizer-storage
YC_STORAGE_REGION=ru-central1

# NextAuth (сгенерируйте случайную строку)
NEXTAUTH_URL=http://localhost:3000
NEXTAUTH_SECRET=любая_длинная_случайная_строка
```

### Шаг 7: Применение миграций

Откройте терминал в папке проекта:

```bash
# Создать миграцию
bunx prisma migrate dev --name init

# Сгенерировать Prisma Client
bunx prisma generate
```

### Шаг 8: Запуск проекта

```bash
bun dev
```

Откройте http://localhost:3000 и попробуйте зарегистрироваться!
